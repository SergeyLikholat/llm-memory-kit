# Настройка под себя

## Главное: категории заводятся сами

**Вам НЕ нужно вручную расписывать свои темы.** Просто работайте с Claude Code как
обычно. Каждую ночь система читает ваши сессии и сама раскладывает активность по темам,
создавая и наращивая карточки активности. «Ремонт велосипеда», «расчёт НДС», «деплой» —
такие темы появляются автоматически из того, о чём вы говорили, а не потому что вы их
завели.

Категории бывают трёх видов:

1. **Темы внутри проекта** — рождаются САМИ из вашей активности. Ничего не делаете.
2. **Проекты** — это ваши рабочие папки `~/projects/<имя>/`. Обычно уже существуют
   (ваши репозитории). Система заводит внутри `memory/` сама при первом прогоне.
   Всё, что вне `~/projects/` (например, работа из домашней папки), идёт в общий контур.
3. **Домены Obsidian** — единственное, что стоит настроить руками, потому что
   структура волта у каждого своя (см. раздел ниже). Только если включили модуль Obsidian.

Всё остальное на этой странице — **опциональная тонкая настройка**. Для старта не нужна:
система работает и без неё, а точность раскладки вы подкрутите позже, когда увидите,
куда что попадает.

## purpose — что относится к проекту

У каждого проекта в `projects/<X>/memory/_index.md` можно задать поле `purpose`:

```yaml
purpose: "Веб-приложение: API, фронтенд, деплой, интеграции"
```

Оно служит двум целям: (1) в главном путеводителе — «за чем сюда идти», (2) для
классификатора — «что сюда складывать». Чем точнее purpose, тем точнее раскладка.

## Рабочие папки проекта

Чтобы папки с файлами (Word/Excel/PDF) всплывали в подсказке при работе по теме —
добавьте секцию в `_index.md` проекта:

```markdown
## 📂 Рабочие директории
- `/path/to/folder` — что там лежит (документы, таблицы)
```

## Домены Obsidian (если включён модуль)

Obsidian-модуль раскладывает ваши заметки по «доменам» — крупным смысловым разделам,
под которые строятся карты (MOC). Структура волта у всех разная, поэтому маппинг
«папка → домен» задаётся вами в `MEMKIT_OBSIDIAN_DOMAINS` (JSON, в `config.sh`):

```bash
export MEMKIT_OBSIDIAN_DOMAINS='{"Projects/":"projects","Areas/":"areas","Resources/":"resources","Inbox/":"inbox"}'
```

- Ключ — префикс пути внутри волта (как называются ваши верхние папки), значение — имя домена.
- Более длинный (более конкретный) префикс имеет приоритет: можно выделить подпапку
  в отдельный домен, например `"Projects/Work/":"work"` поверх общего `"Projects/":"projects"`.
- **Если не задавать** — домен берётся по имени верхней папки автоматически. Работает,
  но домены будут называться как ваши папки.
- Установщик (`install.sh`) при включённом Obsidian покажет ваши верхние папки и поможет
  собрать этот маппинг интерактивно.

## Приватные проекты

Проект можно скрыть из общей карты — перечислите его имя в `MEMKIT_PRIVATE_PROJECTS`
(через запятую, в `config.sh`). В путеводителе он покажется как «приватный контур»,
без раскрытия содержания, а его карточки не индексируются телом для авто-извлечения —
только указатель. По умолчанию список пуст (приватных проектов нет).

## Обслуживание тем

Со временем близкие темы одного проекта могут расплодиться. Раз в несколько недель:

```bash
node core/consolidate.js --project <имя> --dry-run   # показать план слияния
node core/consolidate.js --project <имя>             # слить дубли (с бэкапом)
node core/builder.js --reindex                       # пересобрать оглавления, без LLM
```

`--reindex` не вызывает Claude — пересборка оглавлений мгновенная и бесплатная,
полезна после ручных правок карточек.

## Расписание

Время задаётся в cron: основной прогон 02:30–02:55 + окна-подстраховки 09:00 и 16:00
(на случай, если ночью были исчерпаны лимиты подписки). Частых ретраев нет — обёртки
держат гард (3 попытки/сутки), поэтому лимит не сжигается. Правьте `crontab -e`.
Часовой пояс — в `config.sh` (`MEMKIT_TZ`).

## Ручное управление

- Прогнать сборку разом: `node core/builder.js`
- Только один проект: `node core/builder.js --project имя`
- Проверить без записи: `node core/builder.js --dry-run`
