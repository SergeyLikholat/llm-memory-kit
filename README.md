# 🧠 llm-memory-kit

**Долговременная память для Claude Code.** Ваш AI-ассистент запоминает, над чем вы
работали, накапливает знания по проектам и сам достаёт нужное — без того, чтобы
каждый раз объяснять контекст заново. Работает ночью, автоматически, на вашей подписке.

> Основано на идее [паттерна knowledge base Андрея Карпатого](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f):
> сырьё → аккуратная вики, которая копится и не теряется.

---

## Простыми словами: как это работает

Представьте **помощника с картотекой**:

1. **Днём** он коротко помечает, над чем вы работали (какие задачи, файлы, решения).
2. **Ночью** раскладывает эти пометки по **карточкам-темам** для каждого проекта.
   Карточка не переписывается заново — **дополняется**, поэтому знание копится годами.
3. Есть **тонкое оглавление** — список проектов и тем, без деталей. Только оно всегда
   «на столе» у помощника (не забивает память).
4. Когда вы задаёте вопрос — помощник по оглавлению находит нужную карточку и берёт
   **только её**, не заваливая вас лишним.

Ключевой принцип: **в контекст грузится только лёгкая карта, детали подтягиваются
по запросу.** Чем дольше работаете — тем богаче и полезнее память.

## Что внутри — 3 модуля

| Модуль | Что делает | Обязательный |
|---|---|---|
| **CORE — память сессий** | копит историю работы по проектам, строит карты, авто-извлекает по запросу | ✅ да |
| **OBSIDIAN** | ваши заметки (Obsidian) → авто-обновление вики-карт ночью | ⬜ опционально |
| **SYNC** | синхронизация памяти между сервером, ПК и телефоном | ⬜ опционально |

Можно ставить только CORE, или CORE + OBSIDIAN — на выбор при установке.

## Требования

- **Claude Code** (CLI) с активной подпиской — движок работает на ней, без отдельных API-ключей.
- **Node.js** и **cron** (есть на любом Linux/сервере).
- Опционально: Obsidian-волт (для модуля OBSIDIAN), Syncthing (для SYNC).

## Установка

```bash
git clone https://github.com/SergeyLikholat/llm-memory-kit.git
cd llm-memory-kit
./install.sh
```

Установщик спросит пути, предложит выбрать модули, поставит расписание (cron) и
покажет, что вставить в настройки. Всё по шагам.

## Как устроена память (три хранилища)

```
1. Правила про вас        → MEMORY.md (как обращаться, предпочтения)
2. История по проектам    → projects/<X>/memory/  (карточки-темы + оглавление)
3. Знания из заметок      → Obsidian/_wiki/  (если включён модуль Obsidian)
```

Ночной ритм (по умолчанию, время настраивается):
```
02:30  собрать дневные пометки
02:45  разложить по проектам + построить карточки
02:50  обновить вики из заметок Obsidian
   + повтор каждые 30 мин, если упёрлись в лимит подписки
```

## Настройка под себя

- **`config.sh`** — все пути (создаётся установщиком).
- **`purpose`** у проекта — короткое «что сюда относится» (см. `docs/customize.md`).
  Помогает системе точно раскладывать записи по проектам.
- **Рабочие папки** — привязка папок с файлами к проекту (всплывают в подсказке).
- **Приватность** — проекты можно пометить приватными (не раскрываются в общей карте).

Подробно: **[docs/how-it-works.md](docs/how-it-works.md)** и **[docs/customize.md](docs/customize.md)**.
Синхронизация между устройствами: **[sync/README.md](sync/README.md)**.
